# Agromat Content Analytics

Внутрішня dashboard-аплікація для контент-команди Agromat. Складається з двох інструментів, об'єднаних у вкладки:

1. **Аналіз цін конкурентів** — вбудований через `iframe` legacy Flask-проект `Agromat_Parcer` (на VPS, порт 8080).
2. **Аналіз карток товара** — основний модуль: каталог товарів Agromat зі своєю аналітикою, фільтрами, експортом, snapshot-ами та налаштуваннями обов'язкових атрибутів.

Стек: **Next.js 15 (App Router) · React 19 · TypeScript · TailwindCSS · Upstash Redis · XLSX**.

## Tab 1 — Аналіз цін конкурентів

Файл: `src/app/page.tsx`.

Один `<iframe>` на весь viewport (мінус ~90px під header), що вантажить URL зі змінної `NEXT_PUBLIC_PARCER_URL` (за замовчуванням `http://91.239.233.125:8080/`). Це legacy Flask-додаток `Agromat_Parcer`, що крутиться на VPS і відповідає за моніторинг цін у конкурентів.

- `sandbox` навмисно **не** вказаний — інакше зламаються кукі/POST-форми Flask-аплікації.
- Коли весь стек переїде на один домен з SSL, URL зміниться на щось на кшталт `/parser/` через nginx-proxy.

> Усе, що видно у вкладці, — функціонал зовнішнього застосунку; цей репозиторій його не імплементує.

## Tab 2 — Аналіз карток товара

Цей таб реалізований у `src/components/ProductsCatalog.tsx` (один файл, ~2400 рядків — усі підкомпоненти, модалки, фільтри тримаються разом для зручної навігації). Сторінка `/catalog` просто рендерить `<ProductsCatalog />`.

### Шапка

```
┌─ Картка товару. Каталог. ─────────── [🕐 2026-05-20 ✕] [⚙ Налаштування] ┐
│  [🕐 Перегляд стану каталогу за 2026-05-20 · sync: … · 31 245 товарів]  │  ← банер, якщо обрано snapshot
│                                                                         │
│  [KPI-картки × 8]                                                       │
└─────────────────────────────────────────────────────────────────────────┘
```

| Елемент                              | Що робить при натисканні                                                                                              |
|--------------------------------------|----------------------------------------------------------------------------------------------------------------------|
| Пілюля `🕐 YYYY-MM-DD ✕`            | Видно лише якщо обрано snapshot. Клік → повертає в режим live-даних (`setAsOf(null)`).                              |
| `⚙ Налаштування`                    | Відкриває модалку налаштувань (синхронізація, snapshot-picker, обов'язкові атрибути, звіт останнього синку).        |
| Банер «Перегляд стану…»              | Великий помаранчевий банер у режимі snapshot. Кнопка `✕ Повернутись до поточного` → live-режим.                     |

### KPI-рядок

8 карток, що показують метрики **поточної відфільтрованої вибірки** (число товарів реагує на фільтри). Перша картка додатково показує загальний розмір каталогу у дужках, коли вибірка вже відфільтрована.

| Картка                | Джерело                       |
|-----------------------|-------------------------------|
| Всього товарів        | `data.total` (filtered)       |
| Нових за 24г          | `stats.newCount24h`           |
| Нових за 7д           | `stats.newCount7d`            |
| Зміни статусу 7д      | `stats.statusChanged7d`       |
| Без фото              | `stats.noImages`              |
| Без атрибутів         | `stats.noAttributes`          |
| Без відгуків          | `stats.noReviews`             |
| Без артикулу          | `stats.noSku`                 |

Картки декоративні — клік нічого не робить.

### Зведення по категоріях

Згорнутий за замовчуванням блок (`CategorySummaryPanel`). Дані тягне з `GET /api/products/summary`.

| Елемент                                | Дія при натисканні                                                                                                       |
|----------------------------------------|--------------------------------------------------------------------------------------------------------------------------|
| `Розгорнути / Згорнути`                | Перемикає видимість таблиці; перший раз — лінива загрузка summary.                                                       |
| Input «Пошук категорії…»               | Локальний фільтр за назвою або path категорії.                                                                           |
| Клік на назві категорії                | Drill-down: фільтрує основну таблицю за категорією, статус скидає в дефолтний «В наявності», прокручує до таблиці.       |
| `% з фото / атрибутами / відгуками / артикулом` | Drill-down + одночасно вмикає відповідний фільтр `has_images=false / has_attributes=false / …`.                  |
| `В наявн.` (зелене число)              | Drill-down + `statusIds = [5]`.                                                                                          |
| `Немає` (червоне число)                | Drill-down + `statusIds = [1]`.                                                                                          |
| `Нові товари` (за 7 днів)              | Drill-down + перемикає в пресет-таб «Нові товари» (`tab="new7"`).                                                       |
| `Зміна статусу товара` (за 7 днів)     | Drill-down + перемикає в пресет-таб «Зміна статусу товара» (`tab="changed7"`).                                          |

«Обов'язк. атриб.» — інформаційний стовпчик: показує, скільки атрибутів задано в конфізі обов'язкових для категорії (не клікабельний).

### Пресет-таби + Набір товарів

Над фільтрами розташований ряд:

```
[Всі] [Нові товари] [Зміна статусу товара] [Без фото] [Без атрибутів] [Без відгуків]   [📋 Набір товарів] [✕]
```

| Кнопка                       | Дія                                                                                                          |
|------------------------------|--------------------------------------------------------------------------------------------------------------|
| `Всі`                        | Скидає preset (`tab="all"`).                                                                                 |
| `Нові товари`                | Додає до запиту `only_new_since_sync=true`.                                                                 |
| `Зміна статусу товара`       | Додає `only_status_changed_days=7`; у таблиці з'являється колонка «Було → стало».                            |
| `Без фото / атрибутів / відгуків` | Виставляє `has_images=false` / `has_attributes=false` / `has_reviews=false`.                            |
| `📋 Набір товарів`           | Відкриває `BulkFilterModal` — вставити список ID для точкової фільтрації.                                    |
| `✕` (поряд з «Набір»)        | Скидає активний bulk-фільтр.                                                                                  |

**BulkFilterModal** дозволяє вибрати тип ID (`Код товара` або `goods_ref`), вставити список через будь-який роздільник (пробіли, коми, табуляція, нові рядки) і застосувати фільтр. Парсер дедуплікує і викидає не-числа. Після застосування у таблиці з'являється інформаційний банер з кількістю знайдених / не знайдених і кнопкою «📋 Скопіювати не знайдені».

### Фільтри

Великий рядок керування під табами.

| Контрол                  | Що робить                                                                                                                                               |
|--------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|
| Поле пошуку              | Debounced (300мс). Шукає за назвою, `code`, `sku`, `goods_ref`, `id`.                                                                                  |
| `Всі категорії`          | Searchable single-select (`SearchSelect`). Опції беруться з поточної вибірки (`data.availableCategories`).                                              |
| `Всі бренди`             | Те саме для брендів.                                                                                                                                    |
| `Ціна` (popover)         | Швидкі пресети (до 500, 500–2К, 2К–10К, 10К–50К, 50К+) + ручний діапазон. Live-apply через 300мс debounce. Кнопка `✕ Скинути` всередині.                |
| `Залишок` (popover)      | Кольорові пресети (Немає / Мало / Середн. / Багато / Є) + ручний діапазон. Колір тригера змінюється в залежності від обраного пресету.                  |
| Status-чіпи              | Multi-select: `В наявності`, `Немає в наявності`, `Очікується поставка`, `Під замовлення`, `Знято з виробництва`, `Архів` (UI-only `statusId=-1`).      |
| `Фото: всі / Тільки з фото / Без фото` | Native select для `has_images`.                                                                                                              |
| `Атриб.: …`              | Те саме для `has_attributes`.                                                                                                                           |
| `Відгуки: …`             | Те саме для `has_reviews`.                                                                                                                              |
| `Артикул: …`             | Те саме для `has_sku`.                                                                                                                                  |
| `✕ Скинути` (червона)    | З'являється тільки якщо фільтри відрізняються від дефолту. Повертає все до «В наявності» + sort по `firstSeenAt desc`.                                  |

Дефолтний стан фільтрів: `statusIds=[5]` (В наявності), сорт `firstSeenAt desc`. Зміна будь-якого фільтра автоматично скидає сторінку до `1`.

### Блок «Аналітика» (експорти)

Блакитна панель над таблицею. Усі експорти працюють з **поточно відфільтрованою** вибіркою (внутрішньо тягне до 50 000 рядків через `fetchAllFiltered`).

| Кнопка                       | Дія                                                                                                              |
|------------------------------|------------------------------------------------------------------------------------------------------------------|
| **Goods_ref → `↓ CSV`**      | Завантажує `goods_ref-YYYY-MM-DD.csv` (UTF-8 BOM для Excel).                                                     |
| **Goods_ref → `↓ Excel`**    | Завантажує `goods_ref-YYYY-MM-DD.xlsx`.                                                                          |
| **Goods_ref → `Regex`**      | Копіює в буфер `id1|id2|…`.                                                                                       |
| **IDD (код товару) → `↓ CSV`** | Завантажує `kod-tovara-YYYY-MM-DD.csv`.                                                                       |
| **IDD → `↓ Excel`**          | Завантажує `kod-tovara-YYYY-MM-DD.xlsx`.                                                                          |
| **IDD → `Regex`**            | Копіює `code1|code2|…`.                                                                                           |
| **IDD → `Cube`**             | Копіює MDX-набір: `{ [Товар].[Код товару].&[code1], … }` — для аналітичних кубів.                                |
| **Повна аналітика → `↓ Excel`** | Excel з усіма видимими колонками: код, goods_ref, артикул, назва, категорія, шлях категорії, бренд, ціна, базова ціна, знижка %, валюта, залишок, статус, архів, фото, відгуки, атрибути, рейтинг, URL, всі дати. |

Кожна кнопка після кліку на 1.5 сек показує `✓ Скопійовано`. Поки йде завантаження вибірки — на всіх кнопках відображається `…`.

### Таблиця товарів

Стовпці (у режимі `changed7` додається «Було → стало»):

```
[↗] [Код товара] [goods_ref] [Артикул] [Назва] [Категорія] [Бренд]
[Ціна] [Залишок] [Статус] [Фото] [Відгук.] [Атриб.] [Уперше у нас] [Зміна ст.]
```

**Поведінка рядка:**

| Елемент                  | Дія при натисканні                                                                                                                     |
|--------------------------|----------------------------------------------------------------------------------------------------------------------------------------|
| Сам рядок (будь-де)      | Відкриває `ProductModal` з повною карткою товара.                                                                                       |
| `↗` / `NEW ↗`            | Відкриває URL товара на сайті agromat.ua в новій вкладці. `NEW` — якщо `firstSeenAt` дорівнює часу останнього синку.                   |
| `Код товара`             | Копіює в буфер, тост «Код товара: 1234 — скопійовано».                                                                                  |
| `goods_ref`              | Копіює goods_ref.                                                                                                                       |
| `Артикул`                | Копіює sku.                                                                                                                             |
| `Назва`                  | Як і клік по рядку — відкриває модалку.                                                                                                 |
| `Категорія`              | Виставляє `categoryId` цієї категорії як фільтр.                                                                                       |
| `Бренд`                  | Виставляє `brandId` як фільтр.                                                                                                          |
| `Статус` (пілюля)        | Виставляє `statusIds=[clicked]` (тільки цей статус). Для архівних — встановлює UI-only `statusId=-1`.                                  |
| Заголовки колонок зі стрілками | Перемикають сортування (toggle asc/desc). Активний sort підсвічений синім. Сортовані поля: `name`, `category`, `brand`, `price`, `stockQty`, `imagesCount`, `reviewsCount`, `attributesCount`, `firstSeenAt`, `statusChangedAt`. |

Рядки **нових товарів** мають зелений фон і зелену смужку зліва. Кольори в колонках `Залишок`, `Фото`, `Атриб.` міняються залежно від кількості (зелений — все ОК, помаранчевий — середньо, червоний — нуль/мало).

### Модалка товара

`ProductModal` відкривається кліком на рядку таблиці. Спочатку рендериться seed-копія з таблиці (миттєво), потім підмінюється повним записом з `GET /api/products/:id` (з фото, атрибутами, відгуками, історією статусів).

Розділи:

| Розділ                  | Що показує                                                                                                                                 |
|-------------------------|--------------------------------------------------------------------------------------------------------------------------------------------|
| Identifiers (grid)      | ID, goods_ref, code, sku, бренд, ціна (зі знижкою), залишок, дати, посилання `відкрити →` на agromat.ua.                                  |
| `Фото`                  | Сітка картинок (із зеленим бордером для головного фото). Клік відкриває фото в новій вкладці.                                              |
| `Атрибути`              | Список атрибутів. Атрибути, позначені як обов'язкові для категорії, мають зелений бордер і `★`. Якщо чогось не вистачає — червоний банер.  |
| `Відгуки`               | Картки з рейтингом, перевагами/недоліками, лайками/дізлайками.                                                                              |
| `Історія статусів`      | Хронологічний список переходів статусу (`від → до`, дата/час).                                                                              |

Закриття: кнопка `✕ Закрити`, клік поза модалкою, або клавіша `Esc`.

### Модалка налаштувань

Відкривається з шапки кнопкою `⚙ Налаштування`. Захищена паролем (`Agromat`) — після введення сесія розблокована (зберігається у `sessionStorage`).

Розділи:

| Розділ                       | Контрол                                                                                                                              |
|------------------------------|--------------------------------------------------------------------------------------------------------------------------------------|
| **Синхронізація з API**      | Кнопка `Sync now` → `POST /api/products/sync` з `Bearer NEXT_PUBLIC_DASHBOARD_SECRET`. Поки йде sync — кнопка дизейблиться, з'являється прогрес-бар (`X/Y сторінок · NN% · K тов.`). Опитується `GET /api/products/sync/status` кожні 2 сек. |
| **Перегляд стану на дату**   | `SnapshotPicker` — список доступних snapshot-ів (зберігаються 14 днів). `● Поточний` повертає в live-режим; вибір дати → банер у шапці.  |
| **Обов'язкові атрибути**     | Кнопка `⚙ Відкрити налаштування` → закриває цю модалку і відкриває `RequiredAttrsModal`.                                            |
| Футер «Останній синк»        | Дата останнього успішного синку + статистика (товари / нові / зміни статусу) + посилання `Звіт ↗` на `SyncReportModal`.              |

**SyncReportModal** показує детальний звіт останнього синку: стан, тривалість, старт, кінець, сторінок, товарів, нових, змін статусу + текст помилки (якщо була).

### Обов'язкові атрибути

`RequiredAttrsModal` — двопанельний редактор:

| Панель               | Що в ній                                                                                                                       |
|----------------------|--------------------------------------------------------------------------------------------------------------------------------|
| Ліва — категорії     | Список активних категорій. Зелене число справа = скільки атрибутів уже задано як обов'язкові.                                 |
| Права — атрибути     | При виборі категорії модалка тягне 30 повних карток її товарів і будує список атрибутів за частотою використання. Клік → toggle. |

Кнопка `Зберегти` робить `POST /api/products/required-attrs` з `x-dashboard-secret` заголовком. Конфіг впливає на:

- зелену рамку + `★` у модалці товара;
- червоний банер «Не вистачає обов'язкових атрибутів…» у модалці.

```
├── components/
│   ├── Tabs.tsx                  ← header tab-switcher
│   ├── ProductsCatalog.tsx       ← головний модуль каталогу (усі підкомпоненти inline)
│   └── ui/index.tsx              ← <Card />, базові примітиви
└── lib/
    ├── constants.ts              ← IS_DEV та інші константи
    ├── redis.ts                  ← Upstash Redis client
    ├── products-store.ts         ← read/write нормалізованого каталогу в Redis (типи ProductLite / ProductFull / SyncState)
    ├── products-api.ts           ← клієнт до зовнішнього Agromat API (X-API-Key)
    ├── products-sync.ts          ← оркестрація повного синку
    └── utils.ts                  ← cn() та інші утиліти
```

### Потік даних

```
Зовнішній Agromat API (https://www.agromat.ua/api/v1)
        │
        ▼  products-sync.ts (тригер: cron на VPS або кнопка Sync now)
Upstash Redis
        │  ▲
        │  │
        ▼  │  кеш на 60с в /api/products
Next.js API routes (/api/products/*)
        │
        ▼  fetch + URLSearchParams
ProductsCatalog (React) → користувач
```

Snapshot-и створюються щодня автоматично після успішного синку. Зберігається останні 14 днів.

## API-роути

| Метод   | Роут                                  | Призначення                                                                                                       |
|---------|---------------------------------------|--------------------------------------------------------------------------------------------------------------------|
| GET     | `/api/products`                       | Список товарів з пагінацією, фільтрами, фасетами, статистикою. Основне джерело даних для таблиці.                |
| GET     | `/api/products/:id`                   | Повна картка одного товара (фото, атрибути, відгуки, історія статусів).                                          |
| GET     | `/api/products/filters`               | Дерево категорій + список статусів + список брендів (для початкового рендеру фільтрів).                          |
| GET     | `/api/products/summary`               | Зведена статистика по категоріях для блоку «Зведення по категоріях».                                              |
| GET     | `/api/products/snapshots`             | Список доступних snapshot-дат.                                                                                    |
| GET     | `/api/products/sync/status`           | Поточний стан синку (для polling-у з UI). Завжди bypass кешу.                                                     |
| POST    | `/api/products/sync`                  | Тригерить повний sync. Захищено `Authorization: Bearer ${CRON_SECRET || NEXT_PUBLIC_DASHBOARD_SECRET}`.            |
| GET/POST| `/api/products/required-attrs`        | GET — повертає поточний конфіг. POST — оновлює (потребує заголовок `x-dashboard-secret`).                         |

## Локальний запуск Сантехшари/Vannaja з кнопок dashboard

Кнопки `Сантехшара` і `Vannaja` у блоці цін запускають браузерний парсинг не на VPS, а на ноутбуці користувача. Для цього на ноутбуці має бути запущений local runner:

```bash
cd /path/to/Agromat-Analytics
npm install
npm run local-parser-runner
```

Після цього натискання кнопок у dashboard звертається до `http://127.0.0.1:8765` і runner відкриває локальний термінал з потрібною командою. Для `Vannaja` runner очікує, що репозиторій `Agromat_Parcer` лежить поруч з `Agromat-Analytics`; якщо шлях інший:

```bash
AGROMAT_PARSER_REPO=/absolute/path/to/Agromat_Parcer npm run local-parser-runner
```

## Змінні оточення

Файл `.env.example` (скопіювати в `.env.local`):

```env
# Зовнішній Agromat API
AGROMAT_API_KEY=
AGROMAT_API_BASE_URL=https://www.agromat.ua/api/v1

# Upstash Redis — для каталогу / snapshot-ів / sync state
UPSTASH_REDIS_REST_URL=
UPSTASH_REDIS_REST_TOKEN=

# Bearer для cron на VPS і тригера з UI
CRON_SECRET=

# Захищає мутації з dashboard (required-attrs, sync trigger)
NEXT_PUBLIC_DASHBOARD_SECRET=

# Опціонально — URL legacy парсера, що вантажиться в iframe Tab 1
# NEXT_PUBLIC_PARCER_URL=http://91.239.233.125:8080/
```

> `NEXT_PUBLIC_DASHBOARD_SECRET` потрапляє в клієнтський бандл (це нормально для цього сценарію — секрет використовується лише для розмежування UI-дій, реальна авторизація йде на сервері за `CRON_SECRET`).
